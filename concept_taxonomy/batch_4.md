## 七、高速傳輸 (10 檔 → **全部移除，族群應廢除**)

### 族群問題（最關鍵）
此族群是**「抽象技術概念」**而非實體供應鏈。「高速傳輸」是 SerDes 224G、PCIe Gen6、800G/1.6T 等技術的統稱，不應作為族群分類，應拆解到具體位階。

### 拆解建議

| 代號 | 公司 | 真實位階 | 重分類至 |
|---|---|---|---|
| 2345 智邦 | ODM_SYS | → 網通（核心） |
| 3704 合勤控 | ODM_SYS | → 網通（衛星） |
| 6285 啟碁 | ODM_SYS | → 網通 / 5G（核心） |
| 5388 中磊 | ODM_SYS | → 網通（核心） |
| 3105 穩懋 | FOUNDRY (RF) | → 光通訊/CPO + 5G |
| 3491 昇達科 | OPTIC_COMP | → 5G 基地台 / CPO |
| **3533 嘉澤** | **CONNECTOR** | → 連接器（不是高速傳輸） |
| **3665 貿聯-KY** | **CONNECTOR** | → 連接器（線纜） |
| **2327 國巨** | **PASSIVE** | → 被動元件（MLCC） |
| **6756 威鋒電子** | **IC_DESIGN** | → USB/IC 設計 |

### 結論
**「高速傳輸」族群應從 concept_groups.py 廢除**，10 檔全部已存在於更精確的族群中。保留此族群只會造成位階混淆（嘉澤的 CONNECTOR 被誤認為是「高速傳輸」抽象技術，國巨的 MLCC 被誤認為是高速傳輸主角）。

## 跨族群關鍵發現

### A. 抽象技術 vs 物理元件混淆（最重要）

**「高速傳輸」族群**是最嚴重的位階錯誤範例。10 檔成分股中：
- **3533 嘉澤**：是 `CONNECTOR`（連接器），不是抽象技術。GB300/Rubin 主機板 socket + UQD 液冷快接頭。
- **3665 貿聯-KY**：是 `CONNECTOR`（線纜），高速 cable assembly。
- **2327 國巨**：是 `PASSIVE`（MLCC），高頻被動元件。
- **6756 威鋒**：是 `IC_DESIGN`（USB IC），高速協議晶片。

這些都應依供應鏈位階分類，「高速傳輸」標籤應作為 `core_themes` 之一（如 `SERDES_224G`、`PCIE_GEN6`），而不是族群名稱。

### B. 族群重複問題

| 群組 | 重複族群 | 處理建議 |
|---|---|---|
| 散熱類 | 「散熱/液冷」≈「熱交換器/散熱器」 | 廢除「熱交換器/散熱器」 |
| 光通訊類 | 「光通訊/CPO」≈「矽光子」≈「光通訊細分」 | 「光通訊細分」廢除，併入「LED」 |
| 綠能類 | 「綠能/儲能」≈「儲能系統/BESS」+「太陽能」+「風電」 | 「綠能/儲能」精煉為 EPC + 整合商 |
| 通訊類 | 「網通」≈「5G 通訊」≈「小型通信網路」 | 「小型通信網路」精煉為 IoT 模組 |
| 抽象技術 | 「高速傳輸」 | **廢除**，全部移到具體位階族群 |
